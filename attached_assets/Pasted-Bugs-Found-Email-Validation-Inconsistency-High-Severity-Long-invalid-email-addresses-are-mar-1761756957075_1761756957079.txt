Bugs Found

Email Validation Inconsistency (High Severity)

Long/invalid email addresses are marked as invalid visually (with aria-invalid="true"), but the Submit button is still enabled, allowing invalid emails to be submitted. The UI should disable the Submit button for emails that don’t meet standards.

Misleading 404/Route Handling (Medium Severity)

If you try to visit non-existent routes (like /nonexistent, /leaderboard, /predict, /profile), the app silently redirects you to /splash with no error message or feedback. Instead, a proper 404 message or clear explanation should be shown.

Semantic HTML Structure Issue (Low Severity)

The entire page content is wrapped in a single <p> tag, rather than using semantic elements like <main>, <section>, etc. This is not good practice for accessibility or SEO.

Missing Feedback for ‘Resend Code’ Button (Low Severity)

When using the email confirmation flow, clicking “Resend code” does not provide any confirmation or visible feedback that anything happened.

XSS protection is good (dangerous HTML is escaped)

Valid email addresses—such as those with the + character—are accepted

Proper case-insensitive search for wallet names

Modal overlays, retries, and timeouts work as expected for MetaMask

Recommendations

Fix email validation to block submission of invalid/too-long emails (high priority)

Show proper error/404 messages for invalid routes (medium priority)

Add visual feedback to 'Resend code' button (medium priority)
